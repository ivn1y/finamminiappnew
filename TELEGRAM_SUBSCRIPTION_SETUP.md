# Настройка проверки подписки на Telegram канал

## Что было добавлено

1. **API Endpoint**: `/api/telegram/check-subscription` - проверяет подписку пользователя на Telegram канал
2. **Feature**: `telegram-subscription-check` - хук для проверки подписки в компонентах
3. **Интеграция**: Автоматическая проверка подписки в HomePage

## Требования для работы

### 1. Переменная окружения

**Обязательно** нужно добавить в переменные окружения (`.env.local` для разработки, или в настройках деплоя для продакшена):

```env
TELEGRAM_BOT_TOKEN=ваш_токен_бота
```

### 2. Настройка бота в Telegram

- Бот должен быть добавлен в канал `@finam_collab` как **администратор**
- Бот должен иметь права на просмотр участников канала

### 3. После деплоя

После мерджа и деплоя проверка подписки будет работать автоматически:
- При загрузке страницы HomePage автоматически проверяется подписка
- Если пользователь не подписан, проверка повторяется каждые 5 секунд
- При успешной проверке автоматически обновляется `telegramQuestCompleted` в store

## Как это работает

1. Пользователь открывает страницу с квестом "Выполни первый квест"
2. Автоматически проверяется подписка через Telegram Bot API
3. Если пользователь подписан → квест автоматически отмечается как выполненный
4. Если не подписан → проверка повторяется каждые 5 секунд
5. При клике на кнопку квеста открывается канал, через 2 секунды выполняется повторная проверка

## Технические детали

- Используется Telegram Bot API метод `getChatMember`
- Проверяются статусы: `member`, `administrator`, `creator` (считаются подписанными)
- Статусы `left`, `kicked`, `restricted` (считаются не подписанными)
