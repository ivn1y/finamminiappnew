# 🚀 Локальная версия для тестирования

## 📋 Обзор

Этот проект настроен для локальной разработки и тестирования на компьютере и мобильных устройствах. Включает в себя фронтенд (Next.js) и mock API сервер.

## 🛠️ Быстрый старт

### 1. Установка зависимостей
```bash
npm install
```

### 2. Запуск сервера
```bash
# Простой запуск (рекомендуется)
npm start

# Или через скрипт
./start-universal.sh
```

### 3. Проверка статуса
```bash
npm run status
```

## 📱 Тестирование на разных устройствах

### На компьютере
Откройте браузер и перейдите по адресу:
```
http://localhost:3000
```

### На мобильном устройстве
1. Убедитесь, что телефон и компьютер в одной Wi-Fi сети
2. Откройте браузер на телефоне и перейдите по адресу:
```
http://[IP-адрес-компьютера]:3000
```

### QR код для быстрого доступа
```bash
npm run qr
```

## 📱 Мобильное тестирование

### Требования
1. Мобильное устройство должно быть в той же Wi-Fi сети
2. Компьютер и мобильное устройство должны быть подключены к одному роутеру

### Получение IP-адреса
```bash
# Автоматически (через скрипт)
npm run status

# Вручную
ifconfig | grep "inet " | grep -v 127.0.0.1
```

### QR код для быстрого доступа
```bash
npm run qr
```

## 🛠️ Режимы запуска

### Универсальный режим (рекомендуется)
- Запускает фронтенд с доступом по всем IP
- Автоматически запускает mock API
- Поддерживает hot reload
- Работает на компьютере и мобильных устройствах

### Локальный режим
- Только localhost:3000
- Подходит для разработки на компьютере
- Недоступен с мобильных устройств

### Мобильный режим
- Доступен по IP-адресу компьютера
- Подходит для тестирования на мобильных устройствах
- Требует настройки сети

## 🔧 Настройка сети

### macOS
```bash
# Проверить сетевые интерфейсы
ifconfig

# Проверить брандмауэр
sudo /usr/libexec/ApplicationFirewall/socketfilterfw --getglobalstate
```

### Linux
```bash
# Проверить сетевые интерфейсы
ip addr show

# Проверить брандмауэр
sudo ufw status
```

### Windows
```cmd
# Проверить сетевые интерфейсы
ipconfig

# Проверить брандмауэр
netsh advfirewall show allprofiles
```

## 🐛 Решение проблем

### Порт занят
```bash
# Найти процесс
lsof -i :3000

# Остановить процесс
lsof -ti:3000 | xargs kill -9
```

### Мобильное устройство не подключается
1. Проверьте, что устройства в одной сети
2. Убедитесь, что брандмауэр не блокирует порт 3000
3. Проверьте IP-адрес компьютера
4. Попробуйте отключить VPN

### Страница не загружается
1. Проверьте статус серверов: `npm run status`
2. Проверьте консоль браузера на ошибки
3. Убедитесь, что все зависимости установлены

### API не работает
1. Проверьте, что mock API запущен: `lsof -i :3001`
2. Проверьте доступность: `curl http://localhost:3001/api/health`
3. Перезапустите серверы

## 🔄 Автоматизация

### Автозапуск при изменении файлов
Все серверы поддерживают hot reload и автоматически перезапускаются при изменении кода.

### Автоматическая остановка
При нажатии `Ctrl+C` все серверы корректно останавливаются.

### Автоматическое определение IP
Скрипты автоматически определяют IP-адрес компьютера для мобильного доступа.
