# 🏠 Локальные версии приложения

Простые команды для запуска локальных версий приложения на localhost.

## 🚀 Быстрый запуск

### Локальная версия (localhost)
```bash
npm run local
```
или
```bash
npm run dev:local
```

### Мобильная версия (для тестирования на устройствах)
```bash
npm run mobile
```

## 📋 Доступные команды

| Команда | Описание | URL |
|---------|----------|-----|
| `npm run local` | Локальная версия для разработки | http://localhost:3000 |
| `npm run dev:local` | Альтернативная команда для локальной версии | http://localhost:3000 |
| `npm run mobile` | Мобильная версия для тестирования на устройствах | http://[IP]:3000 |
| `npm run build:local` | Сборка локальной версии | - |
| `npm run start:local` | Запуск собранной локальной версии | http://localhost:3000 |

## 🌐 Доступ к приложению

### Локальная версия
- **URL**: `http://localhost:3000`
- **Назначение**: Разработка и тестирование на компьютере

### Мобильная версия  
- **URL**: `http://[IP_АДРЕС]:3000`
- **Назначение**: Тестирование на мобильных устройствах в локальной сети

## ⚠️ Важные замечания

1. **Порт**: Убедитесь, что порт 3000 свободен
2. **Сеть**: Для мобильного тестирования устройства должны быть в одной Wi-Fi сети
3. **Зависимости**: Убедитесь, что установлены все зависимости: `npm install`

## 🐛 Решение проблем

### Порт занят
```bash
# Проверить, что использует порт 3000
lsof -i :3000

# Остановить процесс (замените PID на реальный)
kill -9 [PID]
```

### Мобильное устройство не подключается
1. Проверьте, что устройства в одной сети
2. Убедитесь, что брандмауэр не блокирует порт 3000
3. Проверьте IP-адрес: `ifconfig` (macOS/Linux) или `ipconfig` (Windows)

## 📞 Поддержка

При возникновении проблем:
1. Проверьте версию Node.js: `node --version` (требуется >=18.0.0)
2. Установите зависимости: `npm install`
3. Очистите кэш: `npm run clean`
