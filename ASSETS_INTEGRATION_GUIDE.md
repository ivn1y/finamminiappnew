## Требования к изображениям

### Персонажи (Characters)
- **Размер**: 512×512px (основной), 1024×1024px (@2x)
- **Формат**: PNG с прозрачным фоном
- **Стиль**: Согласно описаниям из контекста
- **Оптимизация**: Сжатие без потери качества

### Рамки (Frames)
- **Размер**: 200×200px
- **Формат**: PNG или SVG
- **Стиль**: Круглые рамки для аватара
- **Прозрачность**: Центр должен быть прозрачным

### Аксессуары (Accessories)
- **Размер**: 120×120px
- **Формат**: PNG с прозрачным фоном
- **Стиль**: Иконки для наложения на аватар
- **Позиционирование**: Учитывать разные позиции (top, center, bottom, left, right)

## Инструкция по замене

1. **Замените placeholder изображения** в соответствующих папках
2. **Обновите пути** в `src/shared/lib/avatar-assets.ts`:
   ```typescript
   // Замените placeholder пути на реальные
   image: '/assets/avatars/characters/partner.png',
   image2x: '/assets/avatars/characters/partner@2x.png',
   ```

3. **Проверьте fallback** - система автоматически переключится на иконки, если изображения не загрузятся

## Оптимизация производительности

- Используйте WebP формат для лучшего сжатия
- Добавьте lazy loading для изображений
- Реализуйте кэширование изображений
- Используйте responsive images с srcset

## Fallback система

Система автоматически переключается на:
1. **Иконки ролей** - если изображение персонажа не загрузилось
2. **Эмодзи** - для аксессуаров
3. **Градиенты** - для фона аватара

Это обеспечивает работоспособность даже без реальных изображений.
