# 📱 Мобильная версия для тестирования на телефоне

## ✅ ГОТОВО К ИСПОЛЬЗОВАНИЮ

Ваше приложение настроено для тестирования на мобильных устройствах!

## 🚀 Быстрый запуск

### Способ 1: Универсальный скрипт (рекомендуется)
```bash
npm run mobile:universal
# или
./start-mobile-universal.sh
```

### Способ 2: Через npm команды
```bash
npm run mobile
npm run mobile:192
npm run mobile:196
```

## 📱 Доступные адреса для телефона

После запуска сервера ваше приложение будет доступно по следующим адресам:

- **http://192.168.1.13:3000** (ваш текущий IP)
- **http://192.168.1.196:3000** (альтернативный)
- **http://192.168.1.192:3000** (альтернативный)
- **http://localhost:3000** (только на компьютере)

## 📋 Пошаговая инструкция

### 1. Запустите сервер
```bash
npm run mobile:universal
```

### 2. Подключитесь с телефона
1. Убедитесь, что телефон подключен к той же Wi-Fi сети, что и компьютер
2. Откройте браузер на телефоне (Safari, Chrome, Firefox)
3. Введите один из адресов:
   - `http://192.168.1.13:3000`
   - `http://192.168.1.196:3000`
   - `http://192.168.1.192:3000`

### 3. Начните тестирование
Приложение загрузится и будет работать как нативное мобильное приложение!

## 🐛 Решение проблем

### Если не можете подключиться с телефона:

1. **Проверьте Wi-Fi соединение**
   - Убедитесь, что телефон и компьютер в одной сети
   - Проверьте, что Wi-Fi работает стабильно

2. **Проверьте IP-адрес**
   ```bash
   ifconfig | grep "inet " | grep -v 127.0.0.1
   ```

3. **Проверьте брандмауэр**
   - Убедитесь, что порт 3000 не заблокирован
   - На macOS: Системные настройки → Безопасность → Брандмауэр

4. **Попробуйте другой браузер**
   - Safari, Chrome, Firefox
   - Очистите кэш браузера

### Если страница не загружается:

1. **Проверьте статус сервера**
   ```bash
   lsof -i :3000
   ```

2. **Перезапустите сервер**
   - Нажмите `Ctrl+C` в терминале
   - Запустите снова: `npm run mobile:universal`

3. **Проверьте консоль браузера**
   - Откройте инструменты разработчика
   - Посмотрите на ошибки в консоли
