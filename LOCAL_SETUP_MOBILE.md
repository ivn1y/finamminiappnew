# 🚀 Локальные версии приложения

## ✅ Статус: ГОТОВО К ИСПОЛЬЗОВАНИЮ

Ваше приложение настроено и готово к запуску в двух режимах:

## 📱 Мобильная версия (192.168.1.196/192)

### Быстрый запуск
```bash
npm run mobile:196
# или
./start-mobile-196.sh
```

### Доступные адреса
- **Основной**: `http://192.168.1.196:3000`
- **Альтернативный**: `http://192.168.1.192:3000`
- **Автоопределение**: `http://[ваш-ip]:3000`

### Назначение
- Тестирование на мобильных устройствах
- Разработка с поддержкой мобильных функций
- Тестирование в реальных условиях

## 💻 Локальная версия (localhost:3000)

### Быстрый запуск
```bash
npm run localhost
# или
./start-localhost.sh
```

### Доступный адрес
- **URL**: `http://localhost:3000`

### Назначение
- Разработка и отладка на компьютере
- Быстрая итерация кода
- Тестирование десктопной версии

## 🔧 Команды для управления

### Мобильная версия (192.168.1.196)
```bash
# Запуск через npm
npm run mobile:196

# Запуск через скрипт
./start-mobile-196.sh

# Прямой запуск Next.js
npm run dev:mobile
```

### Локальная версия (localhost)
```bash
# Запуск через npm
npm run localhost

# Запуск через скрипт
./start-localhost.sh

# Прямой запуск Next.js
npm run dev:local
```

### Остановка серверов
Нажмите `Ctrl+C` в терминале где запущен сервер

## 📋 Проверка статуса

### Проверить запущенные серверы
```bash
lsof -i :3000
```

### Получить IP-адрес для мобильной версии
```bash
ifconfig | grep "inet " | grep -v 127.0.0.1
```

## 🌐 Доступ к приложению

### На компьютере
Откройте браузер и перейдите по адресу:
```
http://localhost:3000
```

### На мобильном устройстве
1. Убедитесь, что телефон подключен к той же Wi-Fi сети
2. Откройте браузер на телефоне
3. Перейдите по одному из адресов:
```
http://192.168.1.196:3000
http://192.168.1.192:3000
```

## ⚠️ Важные замечания

1. **Порт 3000**: Убедитесь, что порт свободен
2. **Сеть**: Для мобильного тестирования устройства должны быть в одной Wi-Fi сети
3. **IP-адреса**: 192.168.1.196 и 192.168.1.192 - статические адреса для мобильного тестирования
4. **Зависимости**: Убедитесь, что установлены все зависимости: `npm install`

## 🐛 Решение проблем

### Если порт занят
```bash
# Найти процесс использующий порт 3000
lsof -i :3000

# Остановить процесс (замените PID на реальный)
kill -9 [PID]
```

### Если мобильное устройство не подключается
1. Проверьте, что устройства в одной сети
2. Убедитесь, что брандмауэр не блокирует порт 3000
3. Проверьте IP-адрес: `ifconfig` (macOS/Linux) или `ipconfig` (Windows)
4. Попробуйте альтернативный IP: 192.168.1.192

### Если страница не загружается
1. Проверьте, что сервер запущен: `lsof -i :3000`
2. Попробуйте открыть `http://localhost:3000` на компьютере
3. Проверьте консоль браузера на ошибки

## 📞 Поддержка

При возникновении проблем:
1. Проверьте версию Node.js: `node --version` (требуется >=18.0.0)
2. Установите зависимости: `npm install`
3. Очистите кэш: `npm run clean`
