# 🧪 Локальное тестирование

## 🚀 Быстрый старт для тестирования

### 💻 Тестирование на компьютере
```bash
# Запуск локальной тестовой версии
npm run test-local
# или
./test-local.sh
```

**Доступные адреса:**
- Основное приложение: `http://localhost:3000`
- Mock API: `http://localhost:3001`

### 📱 Тестирование на мобильном устройстве
```bash
# Запуск мобильной тестовой версии
npm run test-mobile
# или
./test-mobile.sh
```

**Доступные адреса:**
- Основное приложение: `http://[ваш-ip]:3000`
- Mock API: `http://[ваш-ip]:3001`
- Локальный доступ: `http://localhost:3000`

## 🔧 Что включают тестовые версии

### ✅ Автоматически запускается:
1. **Mock API сервер** (порт 3001) - имитирует бэкенд
2. **Основное приложение** (порт 3000) - ваше приложение
3. **Hot reload** - автоматическая перезагрузка при изменениях

### 📊 Тестовые данные:
- Пользователь с ролью "trader"
- Тестовые QR коды для сканирования
- Контент для изучения
- Система событий и аналитики

## 🎯 Тестовые сценарии

### 1. Регистрация и выбор роли
- Откройте приложение
- Пройдите процесс регистрации
- Выберите роль (trader, expert, partner, startup)

### 2. Настройка профиля
- Заполните данные профиля
- Настройте аватар
- Установите цели на 7 дней

### 3. Тестирование QR сканера
- Используйте тестовые QR коды:
  - `TRADER_001` - для получения бейджа
  - `CONTENT_001` - для разблокировки контента

### 4. Навигация по приложению
- Проверьте все разделы: Home, Chat, Map, Products, Profile
- Тестируйте переходы между экранами
- Проверьте работу форм и интерактивных элементов

## 🛠️ Отладка

### Проверка статуса серверов:
```bash
# Проверить, что слушает порт 3000
lsof -Pi :3000 -sTCP:LISTEN

# Проверить, что слушает порт 3001
lsof -Pi :3001 -sTCP:LISTEN
```

### Остановка всех процессов:
```bash
# Остановить все процессы на портах 3000 и 3001
lsof -ti:3000 | xargs kill -9
lsof -ti:3001 | xargs kill -9
```

### Просмотр логов:
- Логи приложения отображаются в терминале
- Логи Mock API показывают все запросы к API

## 📱 Мобильное тестирование

### Требования:
1. Мобильное устройство должно быть в той же Wi-Fi сети
2. Убедитесь, что файрвол не блокирует порт 3000
3. Для iOS: может потребоваться разрешить HTTP соединения

### Проверка подключения:
```bash
# Узнать IP адрес вашего компьютера
ifconfig | grep "inet " | grep -v 127.0.0.1
```

## 🔍 API Endpoints для тестирования

### Mock API (порт 3001):
- `GET /api/users` - получить всех пользователей
- `GET /api/users/:id` - получить пользователя по ID
- `POST /api/createUser` - создать нового пользователя
- `PUT /api/updateUser/:id` - обновить пользователя
- `POST /api/redeemQR` - использовать QR код
- `POST /api/logEvent` - записать событие
- `GET /api/events/:userId` - получить события пользователя

## ⚠️ Важные замечания

1. **Порты**: Убедитесь, что порты 3000 и 3001 свободны
2. **Мобильное тестирование**: Проверьте подключение к Wi-Fi
3. **Hot Reload**: Изменения в коде автоматически перезагружаются
4. **Остановка**: Используйте Ctrl+C для корректной остановки

## 🆘 Решение проблем

### Порт занят:
```bash
# Найти процесс, использующий порт
lsof -Pi :3000 -sTCP:LISTEN
# Убить процесс
kill -9 [PID]
```

### Мобильное устройство не подключается:
1. Проверьте IP адрес компьютера
2. Убедитесь, что устройства в одной сети
3. Проверьте файрвол
4. Попробуйте перезапустить скрипт

### Mock API не отвечает:
1. Проверьте, что сервер запущен на порту 3001
2. Проверьте логи в терминале
3. Перезапустите скрипт
